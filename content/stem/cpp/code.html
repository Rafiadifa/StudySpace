Every pattern Prof. Ma's TUOJ problems test, organized by category. Each problem has the approach, working code, why it works, and the common pitfalls. The exam mirrors these patterns — recognize the shape and the solution writes itself.

A + B Problem

PATTERN · cin / cout · basic arithmetic · two int read
The pattern

The simplest possible C++ program — read two integers, print their sum. Every exam starts here to warm up. The point is to confirm you can read multiple values from stdin and produce one line of output.

Approach
  1. Include <iostream> and using namespace std;.
  2. Declare two ints (or long long if values may be huge).
  3. Read both with cin >> a >> b; — cin skips whitespace.
  4. Output a + b followed by a newline.
Problem · A + B Contest 63 · #0
Read two integers a and b, output a + b.
#include <iostream>
using namespace std;

int main() {
    int a, b;
    cin >> a >> b;
    cout << a + b << endl;
    return 0;
}
Why it works: cin >> a >> b reads tokens separated by any whitespace (space, tab, newline). endl prints a newline AND flushes the buffer — for competitive problems "\n" is faster but both work.
Watch out: if values can exceed ±2,147,483,647 (about 2×10⁹), use long long not int, or the sum overflows silently.
Problem · a + b (Contest 75) Contest 75 · #4
Same pattern, possibly with a different input format (multiple test cases, or until EOF). The approach is identical.
// If the problem asks for multiple test cases until EOF:
#include <iostream>
using namespace std;

int main() {
    long long a, b;
    while (cin >> a >> b) {
        cout << a + b << "\n";
    }
    return 0;
}
Why while(cin >> a >> b) works: the expression evaluates to the stream itself, which converts to false on EOF or read failure. The loop ends cleanly when input runs out.
Common pitfalls

Forgetting return 0 (gives a warning, not an error in modern compilers but still bad form). Using scanf/printf mixed with cin/cout without ios_base::sync_with_stdio(false) — produces out-of-order output.

Honor Code

PATTERN · string literal output · no input
The pattern

A "Hello World" variant: print a fixed multi-line string. Tests that you can write a literal with cout and handle escape sequences correctly.

Approach
  1. No input. Just print the required text exactly — character for character.
  2. Use "\n" inside the string for line breaks, or chain multiple cout << calls with endl.
  3. Match output spec exactly: a trailing newline is usually required, and double-check capitalization & punctuation.
Problem · Honor Code Contest 63 · #1
Print the university's honor code statement, multiple lines, exact wording.
#include <iostream>
using namespace std;

int main() {
    cout << "I will not cheat on this exam." << endl;
    cout << "I will report any violations I witness." << endl;
    return 0;
}

// Or with a single multi-line string:
// cout << "Line one\nLine two\n";
Why it works: cout just writes its argument to stdout. Multiple chained << operations are evaluated left-to-right, so the lines come out in order.
Watch out: the judge compares output byte-for-byte. A single extra space or missing trailing newline will fail. Copy-paste the expected output if shown.

Life, the Universe, and Everything

PATTERN · loop until sentinel value
The pattern

Read integers one at a time, print each one, but stop when you read 42. This is the standard "loop with sentinel" pattern — process input until you hit a magic terminator value.

Approach
  1. Read one integer at a time inside a while loop.
  2. Before printing, check if it equals the sentinel (42). If yes, break.
  3. Otherwise, print it. Repeat.
Problem · Life, the Universe, and Everything Contest 64 · #0
Read integers, print each, stop when you read 42 (don't print the 42).
#include <iostream>
using namespace std;

int main() {
    int x;
    while (cin >> x) {
        if (x == 42) break;
        cout << x << "\n";
    }
    return 0;
}
Why this order matters: the break must come BEFORE the print, or the 42 would get printed and the loop would still exit. Test on the boundary: the very first input being 42 should produce zero output.

Floating-Point Comparison

PATTERN · epsilon comparison · iomanip formatting
The pattern

Comparing floats with == is broken because of binary representation rounding (e.g. 0.1 + 0.2 != 0.3). The fix is to check whether the difference is smaller than a tiny tolerance ε (epsilon).

Approach
  1. Read the two doubles.
  2. Compute fabs(a - b) from <cmath>.
  3. If the difference is below ε (typically 1e-6 or whatever the problem specifies), they're "equal".
  4. For formatted output use cout << fixed << setprecision(n) from <iomanip>.
Problem · Floating-Point Comparison Contest 63 · #2
Determine whether two real numbers are equal within a given precision.
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    double a, b;
    cin >> a >> b;

    const double EPS = 1e-6;
    if (fabs(a - b) < EPS) {
        cout << "Equal" << endl;
    } else {
        cout << "Not Equal" << endl;
    }
    return 0;
}
Why epsilon works: doubles store ~15–16 significant decimal digits. Operations introduce tiny rounding errors well below 1e-9. Any "real" difference between intended-equal numbers will be smaller than ε, while any "real" inequality will be larger.
Watch out: never use a == b for doubles. Also never test a - b < EPS without absolute value — that misses cases where b > a.

GPA Calculation

PATTERN · weighted average · score → grade mapping
The pattern

Read N courses with their credit hours and percentage scores. Map each percentage to a GPA point (typically 4.0 / 3.7 / 3.3 / 3.0 / ... / 0.0), then compute the credit-weighted average.

Approach
  1. Read N.
  2. For each course: read credit and score. Convert score to GPA point with an if/else chain or a lookup table.
  3. Accumulate totalWeighted += credit * gpaPoint and totalCredits += credit.
  4. Output totalWeighted / totalCredits with the specified precision.
Problem · GPA Contest 64 · #2
Given N courses with credits and scores (0–100), compute the weighted GPA.
#include <iostream>
#include <iomanip>
using namespace std;

double scoreToGPA(int score) {
    if (score >= 90) return 4.0;
    if (score >= 85) return 3.7;
    if (score >= 80) return 3.3;
    if (score >= 75) return 3.0;
    if (score >= 70) return 2.7;
    if (score >= 60) return 2.0;
    return 0.0;
}

int main() {
    int n; cin >> n;
    double totalWeighted = 0, totalCredits = 0;

    for (int i = 0; i < n; i++) {
        int credit, score;
        cin >> credit >> score;
        totalWeighted += credit * scoreToGPA(score);
        totalCredits  += credit;
    }
    cout << fixed << setprecision(2)
         << totalWeighted / totalCredits << endl;
    return 0;
}
Why if-chain in descending order: by checking the highest bracket first, you don't need an upper bound on each if. Once a condition matches, you return and skip the rest. Reversing the order (low to high) would always return 0.0.
Watch out: use the EXACT bracket boundaries the problem gives — different schools use different cutoffs. Read the problem carefully.

Prime Number Judgement

PATTERN · trial division up to √n
The pattern

Decide if N is prime. Brute-force checks every divisor 2..N-1 — that's O(N). The standard optimization is to only check up to √N, because if N = a·b with a ≤ b, then a ≤ √N. That makes it O(√N), fast enough for any exam input.

Approach
  1. Handle base cases: n < 2 is not prime; n == 2 is prime.
  2. If n is even and > 2, not prime.
  3. Loop i from 3 to √n, step 2 (skip even divisors).
  4. If any i divides n evenly, return false. If loop finishes, return true.
Problem · Prime Number Judgement Contest 75 · #0
Read an integer N, print whether it is prime.
#include <iostream>
#include <cmath>
using namespace std;

bool isPrime(int n) {
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false;
    for (int i = 3; i * i <= n; i += 2) {
        if (n % i == 0) return false;
    }
    return true;
}

int main() {
    int n; cin >> n;
    cout << (isPrime(n) ? "Yes" : "No") << endl;
    return 0;
}
Why i * i <= n instead of i <= sqrt(n): it's purely integer math — no floating-point imprecision, no <cmath> required for that line. It's also slightly faster.

Why step by 2: after handling 2 separately, all remaining even numbers are non-prime, so checking them wastes time.

Palindrome Number Judgement

PATTERN · reverse digits via modulo · or string compare
The pattern

Check whether a number reads the same forwards and backwards (121 yes, 123 no). Two approaches: (a) reverse the digits using % 10 and / 10, compare to original; (b) convert to string and compare to its reverse.

Approach (numeric)
  1. Save the original value: int orig = n;
  2. Loop while n > 0: extract last digit with n % 10, append to reversed via reversed = reversed * 10 + (n % 10), then n /= 10.
  3. Return whether reversed == orig.
Problem · Palindrome Number Judgement Contest 75 · #1
Read an integer, check if it reads the same forwards and backwards.
#include <iostream>
using namespace std;

bool isPalindrome(int n) {
    if (n < 0) return false;  // negatives can't be palindromes
    int orig = n, reversed = 0;
    while (n > 0) {
        reversed = reversed * 10 + (n % 10);
        n /= 10;
    }
    return reversed == orig;
}

int main() {
    int n; cin >> n;
    cout << (isPalindrome(n) ? "Yes" : "No") << endl;
    return 0;
}
Why reversed * 10 + digit: each iteration shifts the existing digits left (multiply by 10) and puts the new digit in the ones place. After processing 121: reversed grows 0 → 1 → 12 → 121. Then we compare to the original.

Goldbach's Conjecture

PATTERN · iterate & check pair primality
The pattern

For any even N > 2, find primes p and q such that p + q = N. Loop p from 2 to N/2; if p is prime AND N - p is prime, you've found a valid pair.

Approach
  1. Write a helper isPrime(int) — the trial-division one from the previous topic.
  2. Loop p from 2 to N/2.
  3. If isPrime(p) && isPrime(N - p), output the pair (or count it, depending on the problem variant).
  4. If the problem wants the first such pair, break after finding one.
Problem · Goldbach's Conjecture Contest 64 · #3
Given an even N > 2, find a way to express it as the sum of two primes.
#include <iostream>
using namespace std;

bool isPrime(int n) {
    if (n < 2) return false;
    for (int i = 2; i * i <= n; i++)
        if (n % i == 0) return false;
    return true;
}

int main() {
    int n; cin >> n;
    for (int p = 2; p <= n / 2; p++) {
        if (isPrime(p) && isPrime(n - p)) {
            cout << n << " = " << p << " + " << n - p << endl;
            break;
        }
    }
    return 0;
}
Why loop only to N/2: once p exceeds N/2, q = N - p drops below N/2 — you'd just be finding the same pair in reverse (e.g. 10 = 3+7 then 10 = 7+3). Looping to N/2 cuts the work in half.

Christmas Tree

PATTERN · nested loops · ASCII art
The pattern

Print a triangular pattern of stars (or characters). Outer loop = row, inner loops = spaces and stars. Row i (0-indexed) has n - 1 - i leading spaces and 2i + 1 stars.

Approach
  1. Read height n.
  2. Outer loop: i from 0 to n-1.
  3. Inner loop 1: print n - 1 - i spaces.
  4. Inner loop 2: print 2 * i + 1 stars.
  5. End the row with "\n".
Problem · Christmas Tree Contest 64 · #1
Print a Christmas tree of height N using asterisks.
#include <iostream>
using namespace std;

int main() {
    int n; cin >> n;
    for (int i = 0; i < n; i++) {
        // leading spaces
        for (int j = 0; j < n - 1 - i; j++) cout << ' ';
        // stars
        for (int j = 0; j < 2 * i + 1; j++) cout << '*';
        cout << "\n";
    }
    return 0;
}
Why 2*i + 1 stars: rows widen by 2 each time (1, 3, 5, 7, ...). The pattern is always odd. Combined with n-1-i leading spaces, the apex aligns with the centre.
Watch out: trailing whitespace at end of rows may cause judge failures. Don't add extra spaces AFTER the stars.

Fibonacci Sequence

PATTERN · iterative DP · roll two variables
The pattern

Compute F(n) where F(0)=0, F(1)=1, F(n) = F(n-1) + F(n-2). Recursion is O(2^n) and TLEs. The fix is iterative: keep only the last two values and roll them forward.

Approach
  1. Handle small n: if 0 → 0; if 1 → 1.
  2. Initialize a = 0, b = 1.
  3. Loop from 2 to n: c = a + b; a = b; b = c;.
  4. Output b (the n-th value).
Problem · Fibonacci Sequence Contest 67 · #1
Output the N-th Fibonacci number.
#include <iostream>
using namespace std;

int main() {
    int n; cin >> n;
    if (n == 0) { cout << 0; return 0; }
    if (n == 1) { cout << 1; return 0; }

    long long a = 0, b = 1;
    for (int i = 2; i <= n; i++) {
        long long c = a + b;
        a = b;
        b = c;
    }
    cout << b << endl;
    return 0;
}
Why iterative beats recursive: the naive recursion fib(n) = fib(n-1) + fib(n-2) recomputes the same values exponentially many times. The iterative version computes each value exactly once: O(n) time, O(1) space.
Watch out: F(46) overflows int. F(92) overflows long long. For large n, use long long and check the problem's stated upper bound.

K-th Smallest Number

PATTERN · sort then index · or partial sort
The pattern

Find the K-th smallest element in an array. Simplest robust approach: sort, then index. std::sort from <algorithm> is O(N log N) — fast enough for any exam-sized input.

Approach
  1. Read N and K.
  2. Read N integers into an array or vector<int>.
  3. Call sort(arr, arr + n) or sort(v.begin(), v.end()).
  4. Output element at index k - 1 (since K is 1-indexed in problem, arrays are 0-indexed in code).
Problem · K-th Smallest Number Contest 67 · #2
Given N integers, output the K-th smallest.
#include <iostream>
#include <algorithm>
#include <vector>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;
    vector<int> v(n);
    for (int i = 0; i < n; i++) cin >> v[i];

    sort(v.begin(), v.end());          // ascending
    cout << v[k - 1] << endl;
    return 0;
}
Why k - 1: the problem uses 1-indexed K (the 1st smallest, 2nd smallest, ...), but C++ arrays start at index 0. After sorting ascending, the 1st smallest is v[0], the K-th is v[k-1].
Watch out: if the problem says "K-th LARGEST" instead, either sort descending (sort(v.rbegin(), v.rend())) or use index n - k after ascending sort.

Best Time to Buy and Sell Stock

PATTERN · single-pass tracking · running minimum
The pattern

Given an array of daily prices, find the max profit from one buy + one sell where you must buy before selling. The trick: at each day, the best possible profit if you sell today is price[today] - minPriceSoFar. Track the running minimum and the running best profit in one pass.

Approach
  1. Initialize minPrice = INT_MAX, maxProfit = 0.
  2. For each price:
    • Update minPrice = min(minPrice, price).
    • Update maxProfit = max(maxProfit, price - minPrice).
  3. Output maxProfit.
Problem · Best Time to Buy and Sell Stock Contest 64 · #4
Given prices[N], find max profit from one buy + one sell (buy must come before sell).
#include <iostream>
#include <climits>
#include <algorithm>
using namespace std;

int main() {
    int n; cin >> n;
    int minPrice = INT_MAX, maxProfit = 0;
    for (int i = 0; i < n; i++) {
        int price; cin >> price;
        minPrice  = min(minPrice, price);
        maxProfit = max(maxProfit, price - minPrice);
    }
    cout << maxProfit << endl;
    return 0;
}
Why one pass works: at any day, the best sell-today profit depends only on the minimum price seen in days 0..today. You don't need to store the array — just track the two running values. O(N) time, O(1) space.

Why maxProfit = 0 initially: if prices only go down, no profit is possible, so the answer should be 0 (don't trade), not negative.

Battle Unit Simulator

PATTERN · class with state · methods that mutate
The pattern

The "first real class" problem. Build a class with private state (HP, attack, defense), public methods (attack(), takeDamage(), isAlive()), and a constructor. Simulate a fight or a sequence of actions.

Approach
  1. Define class with private: data members, public: interface.
  2. Constructor takes initial stats via init list: Unit(int h, int a) : hp(h), atk(a) {}.
  3. Method takeDamage(int dmg) subtracts from HP (clamp at 0).
  4. Method isAlive() returns hp > 0 — mark it const.
  5. In main, instantiate units, simulate the turns per the problem spec.
Problem · Battle Unit Simulator Contest 67 · #0
Two units take turns attacking. Output the winner or final state per the rules.
#include <iostream>
#include <string>
using namespace std;

class Unit {
private:
    string name_;
    int hp_, atk_, def_;
public:
    Unit(const string& n, int h, int a, int d)
        : name_(n), hp_(h), atk_(a), def_(d) {}

    bool isAlive() const { return hp_ > 0; }
    int  hp()      const { return hp_; }
    string name()  const { return name_; }

    void attack(Unit& target) {
        int dmg = atk_ - target.def_;
        if (dmg < 0) dmg = 0;
        target.hp_ -= dmg;
        if (target.hp_ < 0) target.hp_ = 0;
    }
};

int main() {
    Unit a("A", 100, 20, 5);
    Unit b("B", 100, 15, 3);
    while (a.isAlive() && b.isAlive()) {
        a.attack(b);
        if (b.isAlive()) b.attack(a);
    }
    cout << (a.isAlive() ? a.name() : b.name()) << " wins" << endl;
    return 0;
}
Why attack() can access target.def_: member functions of class Unit can access private members of any Unit object, not just this. Access control is per-class, not per-instance.

Why const on getters: declares that the method doesn't modify state. Allows the method to be called on const Unit& references too.

Polymorphic Areas

PATTERN · abstract base · pure virtual · override · array of base pointers
The pattern

The polymorphism test. Define an abstract Shape with pure virtual area(), derive Circle, Rectangle, etc. Store them in an array of Shape*, loop and sum their areas — virtual dispatch picks the right area() at runtime.

Approach
  1. Base class Shape: virtual double area() = 0; + virtual ~Shape() = default;.
  2. Each subclass: store its dimensions, implement area() override.
  3. In main: Shape* shapes[N];, populate with new Circle(...) etc.
  4. Loop: total += shapes[i]->area(); — calls the correct override.
  5. Loop again to delete shapes[i] — works correctly because the destructor is virtual.
Problem · Polymorphic Areas Contest 75 · #2
Given a list of shapes (circles, rectangles, triangles, ...), compute the sum of their areas using polymorphism.
#include <iostream>
#include <vector>
#include <iomanip>
using namespace std;

class Shape {
public:
    virtual double area() const = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14159265 * r * r; }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double area() const override { return w * h; }
};

int main() {
    vector<Shape*> shapes;
    shapes.push_back(new Circle(5));
    shapes.push_back(new Rectangle(3, 4));

    double total = 0;
    for (Shape* s : shapes) total += s->area();

    cout << fixed << setprecision(2) << total << endl;

    for (Shape* s : shapes) delete s;
    return 0;
}
Why dynamic dispatch works: because area() is virtual, each Shape* secretly stores a pointer to its real type's vtable. s->area() looks up the right function at runtime — Circle's for Circle objects, Rectangle's for Rectangle objects.

Why virtual destructor: delete s with a Shape* would only call ~Shape() if the destructor weren't virtual — Circle's destructor would be skipped, leaking r. With virtual ~Shape(), the chain runs correctly: ~Circle() first, then ~Shape().
Watch out: forgetting override hides typos — if you write area() without const when the base has const, you've defined a new function instead of overriding. override catches this at compile time.

Friend Function

PATTERN · friend declaration · access to private members
The pattern

Sometimes a non-member function needs access to a class's private data — most commonly operator overloads like operator<<, or functions that need to inspect two different classes. Declare it friend inside the class.

Approach
  1. Inside the class, declare friend on the function prototype (in any access section — it doesn't matter).
  2. Outside the class, define the function normally — no friend keyword on the definition, no class scope (ClassName::) since friends aren't members.
  3. The function can now read/write any private member.
Problem · Friend Function Contest 75 · #3
Implement a class with private data and a friend function that prints / manipulates it.
#include <iostream>
using namespace std;

class Box {
private:
    int width, height;
public:
    Box(int w, int h) : width(w), height(h) {}

    // Declare friend — grants access to private members
    friend int area(const Box& b);
    friend ostream& operator<<(ostream& os, const Box& b);
};

// Definition — NO Box:: scope, NO friend keyword here
int area(const Box& b) {
    return b.width * b.height;   // access private via friendship
}

ostream& operator<<(ostream& os, const Box& b) {
    os << "Box(" << b.width << "x" << b.height << ")";
    return os;
}

int main() {
    Box b(3, 4);
    cout << b << " area = " << area(b) << endl;
    return 0;
}
Why operator<< must be a friend (not a member): the left operand is ostream, not Box. If it were a member of Box, you'd have to write b << cout — backwards. Making it a free function with friendship lets the natural cout << b syntax work AND lets it read Box's private fields.
Watch out: friendship is not inherited (a derived class's friends aren't friends of the base) and not transitive (A's friend isn't automatically B's friend). Use it sparingly — it breaks encapsulation.

One-Move Chess Problem

PATTERN · class hierarchy · virtual move validation
The pattern

Combines class design + polymorphism. Base class Piece with abstract canMove(from, to); subclasses (Pawn, Rook, Knight, ...) each define their own movement rules. Given a board and a move, determine if it's legal.

Approach
  1. Abstract Piece with virtual bool canMove(int fr, int fc, int tr, int tc) const = 0;
  2. Each subclass implements its rule — Rook: same row OR same col; Bishop: |dr| == |dc|; Knight: (|dr|,|dc|) = (1,2) or (2,1); etc.
  3. Store pieces in Piece* board[8][8] or similar.
  4. Look up the piece at the source square, call its canMove().
Problem · One-Move Chess Contest 67 · #3
Given a piece type and a from/to coordinate pair, decide whether the move is legal in chess rules.
#include <iostream>
#include <cmath>
using namespace std;

class Piece {
public:
    virtual bool canMove(int fr, int fc, int tr, int tc) const = 0;
    virtual ~Piece() = default;
};

class Rook : public Piece {
public:
    bool canMove(int fr, int fc, int tr, int tc) const override {
        return (fr == tr || fc == tc) && !(fr == tr && fc == tc);
    }
};

class Bishop : public Piece {
public:
    bool canMove(int fr, int fc, int tr, int tc) const override {
        return abs(fr - tr) == abs(fc - tc) && fr != tr;
    }
};

class Knight : public Piece {
public:
    bool canMove(int fr, int fc, int tr, int tc) const override {
        int dr = abs(fr - tr), dc = abs(fc - tc);
        return (dr == 2 && dc == 1) || (dr == 1 && dc == 2);
    }
};

int main() {
    string type;
    int fr, fc, tr, tc;
    cin >> type >> fr >> fc >> tr >> tc;

    Piece* p = nullptr;
    if      (type == "rook")   p = new Rook();
    else if (type == "bishop") p = new Bishop();
    else if (type == "knight") p = new Knight();

    cout << (p && p->canMove(fr, fc, tr, tc) ? "Yes" : "No") << endl;
    delete p;
    return 0;
}
Why polymorphism over a switch: if you used a giant switch(type) statement instead, every new piece type would require editing the switch. With polymorphism, adding a Queen is just adding a new class — no existing code changes. This is "open for extension, closed for modification."

Why the Rook check excludes fr == tr && fc == tc: "moving to the same square" isn't a real move. Always exclude the zero-move case unless the problem allows it.
