Inheritance & UML Midterm Review

Fast, beginner-friendly English review for students with about 2 months of C++ study.

A. Exam Focus in 60 Seconds

  • Know the difference between is-a (inheritance) and has-a (composition / aggregation).
  • Remember access control: public / protected / private and the friend exception.
  • Master constructor and destructor sequence: base first, reverse on destruction.
  • Use initialization lists correctly: call base constructor and initialize members.
  • Understand public/protected/private inheritance effects on base members.
  • Recognize UML symbols: + - #, hollow triangle, diamond types.

B. Concept Cheat Sheets

Common trap: Private inheritance is not equivalent to composition in design meaning.

Construction: Base constructors -> member objects -> derived constructor body.

Destruction: Reverse order of construction.

Memory tip: Build from ground up; tear down from roof down.
  • friend function: a non-member that can access private/protected members.
  • friend class: all methods in that class get access.
  • Friendship is not inherited and not transitive.
Common trap: “friend” does not make the function a member function.

C. Must-Know Code Templates

Public inheritance + initialization list

class Animal {
public:
    Animal(const string& n) : name(n) {}
protected:
    string name;
};

class Dog : public Animal {
public:
    Dog(const string& n, int a)
        : Animal(n), age(a) {}
private:
    int age;
};

Friend function

class Box {
private:
    int secret = 42;
    friend void reveal(const Box& b);
};

void reveal(const Box& b) {
    cout << b.secret << endl;
}

Virtual base for diamond problem

class A { public: int x = 1; };
class B : virtual public A {};
class C : virtual public A {};
class D : public B, public C {};

// D has only one A subobject.

C2. static / const / static const (Quick Review)

Keyword Core Meaning Where Stored / Shared Exam Trap
static member Belongs to class, not each object One shared copy for all objects Must define it outside class (for non-inline form)
const member Value cannot change after initialization Stored in each object Must initialize in constructor init list
static const member Shared constant at class level One class-level constant value Confusing it with object-level const member

Minimal Example

class Student {
public:
    Student(int id) : id_(id) { ++count_; }

    static int getCount() { return count_; }
    int getId() const { return id_; }

    static const int maxScore = 100;

private:
    static int count_;
    const int id_;
};

int Student::count_ = 0;
Fast memory rule: static = one shared box, const = locked value, static const = shared locked value.
Common mistakes: forgetting init list for const member, and forgetting external definition for static data member.

D. Hard Parts & Common Mistakes

Hard Point 1: Confusing actual initialization order with the order written in initialization list. Real rule: member initialization follows declaration order in class definition.
Hard Point 2: Thinking derived constructor body runs before base constructor. Real rule: base construction always happens first.
Hard Point 3: Using private inheritance when the model should be “has-a”. Use composition if ownership/object containment is the real design.
Quick anti-mistake checklist: confirm relationship type, confirm access level, trace constructor chain.

E. Practice Zone

Q1. Access Control Judgment Basic

In public inheritance, can a derived method directly use base private members?

Answer: No.

Why: Private members stay accessible only inside the base class itself.

Memory Tip: Private = bedroom, children cannot enter directly.

Q2. Constructor Order Core

Given class D : public B with member object M m;, choose order:

A) D body -> B -> M
B) B -> M -> D body
C) M -> B -> D body

Answer: B) B -> M -> D body

Memory Tip: Foundation first, decoration later.

Q3. Code Correction Core

Fix the derived constructor:

class Base {
public:
    Base(int id) {}
};
class Derived : public Base {
public:
    Derived(int id, int score) { /* ... */ } // wrong
};
class Derived : public Base {
public:
    Derived(int id, int score)
        : Base(id), score_(score) {}
private:
    int score_;
};

Why: Base has no default constructor, so it must be called in init list.

Q4. UML Mapping Basic

“Car has Engine, and Engine lifetime is controlled by Car.” Which relation?

Answer: Composition (strong has-a, solid diamond).

Q5. Friend Concept Core

True or False: if class A is friend of B, all subclasses of A are automatically friend of B.

Answer: False.

Why: Friendship is not inherited.

Q6. Diamond Problem Challenge

Why use virtual inheritance in diamond structures?

Answer: To keep only one shared base subobject and remove ambiguity / duplication.

class B : virtual public A {};
class C : virtual public A {};
class D : public B, public C {};

Q7. static vs const vs static const Core

Choose the correct statement:

A) const data member can be assigned in constructor body.
B) static data member has one copy per object.
C) static const expresses a class-level constant.
D) static member can only be accessed by object.

Answer: C is correct.

Why: const member needs init list; static is shared by class and can be accessed with ClassName::member.
