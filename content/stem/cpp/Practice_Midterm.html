Q4 Core Scope & Variables

What is the output of the following code?

int x = 10;
{
    int x = 20;
    cout << x << " ";
}
cout << x;
A10 10
B20 20
C20 10
DCompile error: redefinition
The inner block creates a new x that shadows the outer one. When the block ends, the outer x (=10) is restored.

Q6 Core OOP — Access Specifiers

Q7 Core Constructors

int main() {
    Dog d;
    cout << "Alive ";
}
AAlive Born Gone
BBorn Alive Gone
CBorn Gone Alive
DAlive Gone Born
Constructor runs when the object is created (before main body continues). Destructor runs when the object goes out of scope (end of main).
Q8 Core Inheritance

In C++ public inheritance, which members of the base class become private in the derived class?

AThe base's public members
BThe base's protected members
CThe base's private members (they remain inaccessible, not inherited directly)
DNone — everything becomes public
With public inheritance: public→public, protected→protected. Private members of the base are never directly accessible in derived classes.

Q10 Challenge Function Overloading

Q13 Core Inheritance — Constructor Order

What is the output?

class A {
public:
    A() { cout << "A "; }
    ~A() { cout << "~A "; }
};
class B : public A {
public:
    B() { cout << "B "; }
    ~B() { cout << "~B "; }
};
int main() {
    B obj;
}
Base constructors run before derived constructors. Destructors run in reverse order.

Q15 Core Bug Identification

int arr[] = {3, 9, 2, 7, 1};
int maxVal = 0;
int maxIdx = 0;
for (int i = 0; i < 5; i++) {
    if (arr[i] > maxVal) {
        maxVal = arr[i];
        maxIdx = i;
    }
}
cout << "Max: " << maxVal << " at index " << maxIdx;
The code works here, but think: what if all values were negative? Initializing maxVal = 0 would be wrong then. For this array, does it give the right answer?

    }
