Functions, Classes & Objects Midterm Review

Fast, beginner-friendly English review for Chapter 4: Functions, Classes, Objects, and Multi-file Design.

A. Exam Focus in 60 Seconds

  • Master function definition: return type, name, parameters, body.
  • Understand pass by value (copy) vs pass by reference (& modifies original).
  • Know function overloading rules: different parameter types/count required.
  • Separate declaration (.h) from definition (.cpp) for organization.
  • Class = blueprint; Object = instance. Member access uses . operator.
  • Use header guards (#ifndef/#define/#endif) to prevent double inclusion.
  • Apply encapsulation: private data + public getters/setters for validation.

E. Practice Problems

Q1. Find the Bug Basic

What's wrong with this code?

class Point {
public:
    int x;
    int y;
    void print()
}

int main() {
    Point p();
    p.x = 5;
    p.print();
}

Three errors:

  1. Missing ; after print() declaration inside class
  2. Missing ; after closing brace of class
  3. Point p(); declares a function, not an object. Should be Point p;
// Corrected:
class Point {
public:
    int x;
    int y;
    void print();  // ← semicolon
};               // ← semicolon

void Point::print() {  // Implementation
    cout << "(" << x << "," << y << ")" << endl;
}

int main() {
    Point p;     // ← no parentheses!
    p.x = 5;
    p.print();
}

Q2. Pass by Value vs Reference Core

What is the output?

void doubleValue(int x) {
    x = x * 2;
    cout << "Inside: " << x << endl;
}

void doubleRef(int& x) {
    x = x * 2;
    cout << "Inside: " << x << endl;
}

int main() {
    int a = 5, b = 5;
    doubleValue(a);
    doubleRef(b);
    cout << "a = " << a << ", b = " << b << endl;
}
Inside: 10    // doubleValue - local copy doubled
Inside: 10    // doubleRef - original doubled
a = 5, b = 10 // a unchanged (pass by value), b changed (pass by reference)
Key: Pass by value creates a copy. Pass by reference uses the original variable.

Q3. Overloading Validity Core

Which of these overloads are valid?

A. int calc(int x);
   double calc(int x);

B. int calc(int x);
   int calc(double x);

C. int calc(int x, int y);
   int calc(int x);
  • A: INVALID — only return type differs
  • B: VALID — parameter type differs
  • C: VALID — parameter count differs
Rule: Overloading requires different parameter types, count, or order. Return type alone is not enough.

class BankAccount {
private:
    int accountNumber;
    double balance;
    
public:
    // Setter with validation
    void setBalance(double amount) {
        if (amount >= 0) {
            balance = amount;
        } else {
            cout << "Error: Balance cannot be negative!" << endl;
        }
    }
    
    // Getter
    double getBalance() const {
        return balance;
    }
    
    // Withdraw with validation
    bool withdraw(double amount) {
        if (amount <= balance) {
            balance -= amount;
            return true;  // Success
        }
        cout << "Insufficient funds!" << endl;
        return false;  // Failed
    }
    
    // Initialize
    void initialize(int num, double bal) {
        accountNumber = num;
        setBalance(bal);  // Use setter for validation
    }
};

Q6. Complete Multi-File Project Challenge

Create a Rectangle class with proper multi-file organization:

  • rectangle.h — Class declaration with width, height (private), getArea(), getPerimeter()
  • rectangle.cpp — Implementations
  • main.cpp — Create Rectangle objects and print area and perimeter

rectangle.h:

#ifndef RECTANGLE_H
#define RECTANGLE_H

class Rectangle {
private:
    double width;
    double height;
    
public:
    void setDimensions(double w, double h);
    double getArea() const;
    double getPerimeter() const;
};

#endif

rectangle.cpp:

#include "rectangle.h"

void Rectangle::setDimensions(double w, double h) {
    if (w > 0 && h > 0) {
        width = w;
        height = h;
    }
}

double Rectangle::getArea() const {
    return width * height;
}

double Rectangle::getPerimeter() const {
    return 2 * (width + height);
}

main.cpp:

#include <iostream>
#include "rectangle.h"
using namespace std;

int main() {
    Rectangle r;
    r.setDimensions(5, 3);
    cout << "Area: " << r.getArea() << endl;
    cout << "Perimeter: " << r.getPerimeter() << endl;
    return 0;
}

// Compile: g++ rectangle.cpp main.cpp -o rectangle_test

Q7. Predict Output Challenge Challenge

What does this print? Trace carefully!

class Counter {
    int id;
    static int total;
public:
    Counter(int i) : id(i) {
        total++;
        cout << "Created #" << id << " (total: " << total << ")" << endl;
    }
    ~Counter() {
        total--;
        cout << "Destroyed #" << id << " (total: " << total << ")" << endl;
    }
};
int Counter::total = 0;

void test() {
    Counter c1(1);
    Counter c2(2);
}

int main() {
    cout << "Start" << endl;
    test();
    cout << "Middle" << endl;
    test();
    cout << "End" << endl;
    return 0;
}
Start
Created #1 (total: 1)
Created #2 (total: 2)
Destroyed #2 (total: 1)
Destroyed #1 (total: 0)
Middle
Created #1 (total: 1)
Created #2 (total: 2)
Destroyed #2 (total: 1)
Destroyed #1 (total: 0)
End
Key insights:
  • Constructors called in order of declaration
  • Destructors called in reverse order (LIFO)
  • Local objects destroyed when leaving scope
  • Static member total shared across all objects
