Encapsulation & Operator Overloading
C++ Final10 June 2026
Two ways to overload. As a member: T operator+(const T& rhs) const — the left operand is *this. As a non-member / friend: T operator+(const T& a, const T& b) — needed when the left operand isn't your class (e.g. operator<< with ostream on the left). Return by value for arithmetic, by reference (ostream&) for streaming.
Problem 1 Why encapsulate? Concept
A teammate makes balance public "to keep it simple." Give two concrete bugs this invites, and rewrite the class to prevent them.
Answer (1) Anyone can set balance = -9999, breaking the invariant "balance ≥ 0". (2) If you later change how balance is stored (cents vs dollars), every external caller breaks. Encapsulation fixes both — the invariant is enforced in one place and the representation can change freely. 
class Account {
    double balance = 0;            // private: invariant guarded
public:
    bool withdraw(double a) {
        if (a <= 0 || a > balance) return false;
        balance -= a; return true;
    }
    double get() const { return balance; }
};
Problem 2 Overload operator+ (member) Member
Give Vector2D{x,y} a member operator+ so that a + b adds componentwise. Why should it be const and return by value?
Answer 
class Vector2D {
    double x, y;
public:
    Vector2D(double x=0, double y=0) : x(x), y(y) {}
    Vector2D operator+(const Vector2D& r) const {
        return Vector2D(x + r.x, y + r.y);
    }
};
const — adding two vectors mustn't mutate either. Return by value — the sum is a brand-new vector, not a reference to an existing one (returning a reference to a local is a dangling-reference bug).
Problem 3 Overload operator<< (friend) Non-member
Make cout << v print (x, y). Why can't this be a member function?
Answer The left operand of << is an ostream, not your class — so it can't be a member of Vector2D. Make it a non-member (a friend so it can read private x,y), take ostream& and return it to allow chaining. 
friend ostream& operator<<(ostream& os, const Vector2D& v) {
    os << "(" << v.x << ", " << v.y << ")";
    return os;          // return ostream& so cout << a << b chains
}
Returning os by reference is what makes cout << v1 << v2 << "\n" work — each << hands the same stream to the next.
Problem 4 operator== and operator* Apply
Add equality and a scalar product: v == w and v * 3.0 (scale both components). Write both signatures.
Answer 
bool operator==(const Vector2D& r) const {
    return x == r.x && y == r.y;
}
Vector2D operator*(double s) const {
    return Vector2D(x * s, y * s);
}
Note v * 3.0 works as a member (left operand is the vector), but 3.0 * v would need a non-member overload with the scalar on the left. Mentioning that asymmetry scores the conceptual point.
