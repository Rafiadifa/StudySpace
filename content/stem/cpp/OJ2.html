Problem 1 Fibonacci Queries Precompute
With \(f_0=f_1=1\) and \(f_i=f_{i-1}+f_{i-2}\), answer \(T\) queries each asking for \(f_n\) (with \(n\le80\)).
Input
3
0
10
80
Output
1
89
23416728348467685
Approach \(f_{80}\) overflows 32-bit int, so use long long. Precompute the whole table once up to 80, then each query is an \(O(1)\) lookup. 
long long f[81];
f[0] = f[1] = 1;
for (int i = 2; i <= 80; i++) f[i] = f[i-1] + f[i-2];

int T; cin >> T;
while (T--) { int n; cin >> n; cout << f[n] << "\n"; }
Precomputing beats recomputing per query: \(T\) queries cost \(O(T)\), not \(O(T\cdot n)\). The long long is the trap most people miss.
Problem 2 RPG Battle Simulator Polymorphism
Build a battle system. Units share a base with name, hp, maxHp, atk. Commands: CREATE type name hp atk, ATTACK a b, STATUS name (print hp), ALIVE name (print YES/NO).

Class rules: Warrior — deals atk. Archer — deals atk+10 if target hp > 50, else atk. Mage — deals atk+15 but takes 5 recoil. Tank — deals atk, but only takes 70% of incoming damage. Healer — "attack" instead heals the target by atk (capped at maxHp).
Design An abstract base Unit with a pure virtual attack() and a virtual takeDamage(). Each class overrides attack() for its rule; only Tank overrides takeDamage() to apply the 70% reduction before delegating to Unit::takeDamage. 
class Unit {
protected:
    string name; int hp, maxHp, atk;
public:
    Unit(string n, int hp, int atk)
        : name(n), hp(hp), maxHp(hp), atk(atk) {}
    virtual ~Unit() = default;
    bool isAlive() const { return hp > 0; }
    virtual void attack(Unit& t) = 0;        // pure virtual
    virtual void takeDamage(int d) {
        if (!isAlive()) return;
        hp -= d; if (hp < 0) hp = 0;
    }
    void heal(int v) { if (isAlive()) hp = min(maxHp, hp + v); }
};
The five overrides 
void Archer::attack(Unit& t) {
    if (t.getHp() > 50) t.takeDamage(atk + 10);
    else t.takeDamage(atk);
}
void Mage::attack(Unit& t) { t.takeDamage(atk + 15); takeDamage(5); }
void Healer::attack(Unit& t) { t.heal(atk); }
// Tank overrides takeDamage, not the rule for attack
void Tank::takeDamage(int d) { Unit::takeDamage((d * 7) / 10); }
Store units in a unordered_map<string, unique_ptr<Unit>> keyed by name. The dispatcher just calls attacker->attack(*target) — the right override fires automatically via the vtable. Always guard "is the attacker/target alive?" before acting.
Problem 3 k-th Smallest Element Sort
Read \(n\) and \(k\), then \(n\) integers. Print the \(k\)-th smallest value (1-indexed).
Input
5 2
9 3 7 1 5
Output
3
Approach Sort ascending, then the \(k\)-th smallest is arr[k-1]. Simple and clean with std::sort. 
vector<int> arr(n);
for (int& x : arr) cin >> x;
sort(arr.begin(), arr.end());
cout << arr[k - 1];
The k-1 shift converts 1-indexed "k-th" to 0-indexed array access. For huge \(n\), nth_element gives \(O(n)\) average, but sorting is fine here.
