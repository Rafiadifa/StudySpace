Q5 Hard Virtual · Polymorphism

What is the output?

class Shape { public: void draw() { cout << "Shape "; } virtual void area() { cout << "S-area "; } }; class Circle : public Shape { public: void draw() { cout << "Circle "; } void area() { cout << "C-area "; } }; int main() { Shape* p = new Circle(); p->draw(); p->area(); delete p; }
ACircle C-area
BShape C-area
CCircle S-area
DShape S-area
draw() is NOT virtual — so the pointer type (Shape*) decides which version runs. area() IS virtual — so the actual object type (Circle) decides. This is the core of polymorphism.
Q6 Expert Diamond Problem · Virtual Inheritance

Without virtual inheritance, how many copies of A's data does object d contain, and what is the problem with d.show()?

class A { public: int x=1; void show(){cout<<x;} }; class B : public A {}; class C : public A {}; class D : public B, public C {}; int main() { D d; d.show(); // what happens? }
A1 copy of A; d.show() compiles and prints 1
B2 copies of A; d.show() is ambiguous — compile error
C2 copies of A; d.show() picks one randomly
D0 copies of A; D cannot inherit A indirectly
Without virtual inheritance, B carries its own A subobject and C carries its own A subobject. D inherits both → 2 copies → ambiguity. Fix: class B : virtual public A and class C : virtual public A.
Q7 Hard Overloading vs Overriding

Q10 Expert const Correctness

Q12 Expert UML · Relationships

A Hospital creates Room objects on construction and destroys them when it closes. A Doctor works at a Hospital but can exist independently. Which pair of UML relationships is correct?

AHospital–Room: Aggregation (hollow ◇)  |  Hospital–Doctor: Composition (solid ◆)
BHospital–Room: Composition (solid ◆)  |  Hospital–Doctor: Aggregation (hollow ◇)
CBoth are Composition

Q15 Hard Operator Precedence · Boolean Logic

Q17 Expert Virtual Inheritance · Diamond Fix

Fix the diamond problem so D has only one copy of A. Fill in both blanks (they are the same keyword).

class A { public: int x; }; class B : A {}; class C : A {}; class D : public B, public C {}; // Now D has only 1 copy of A
Both B and C must inherit from A using virtual inheritance: class B : public virtual A {}; — this tells the compiler to share a single A subobject in any diamond descendant.
