C. Must-Know Code Templates

1) Safe variable use + address check

int age = 25;
cout << "Value: " << age << endl;
cout << "Address: " << &age << endl;

2) Floating-point comparison (epsilon)

#include 
double a = 0.1 + 0.2;
if (fabs(a - 0.3) < 1e-9) {
    cout << "Equal enough" << endl;
}

3) Function with parameters and return value

double average(int a, int b, int c) {
    return (a + b + c) / 3.0;
}

int main() {
    int x, y, z;
    cin >> x >> y >> z;
    cout << average(x, y, z) << endl;
    return 0;
}

D. Program Flow Diagram (Input -> Process -> Output)

Execution thinking: declare variables -> read input with cin -> compute by expressions/functions -> print with cout.

Quality check: ensure type match, division type, and precedence clarity before output.

Memory tip: "Collect ingredients -> cook -> plate the dish."

E. Hard Points & Common Mistakes

Hard Point 1: Confusing declaration with initialization. Declaration alone does not assign safe value.
Hard Point 2: Forgetting integer division behavior in mixed calculations.
Hard Point 3: Comparing doubles with == directly.
Hard Point 4: Misreading operator precedence when there are multiple operators.
Anti-mistake checklist: initialize, check operand types, add parentheses, test with sample input.

F. Practice Zone (Click to Reveal)

Q1. Declaration vs Initialization Basic

Which line is initialization?

A) int score; B) score = 90; C) int score = 90;

Answer: C

Why: It creates variable and gives first value in one statement.

Memory Tip: [Rule] First value at birth = initialization.

Q2. Uninitialized Risk Core

What is wrong with int x; cout << x;?

Answer: x is uninitialized and may print random garbage.

Why: This is undefined behavior in C++.

Memory Tip: [Analogy] Unlabeled box content is unknown - never trust it.

Q3. Division Trap Core

What is the output of double d = 5 / 2; cout << d;?

Answer: 2 (or 2.0)

Why: 5 / 2 is integer division first, then converted to double.

Memory Tip: [Rule] Integer + integer division => truncation first.

Q4. Precedence Basic

Evaluate: 10 + 5 * 2 and (10 + 5) * 2.

Answer: 20 and 30.

Why: Multiplication has higher precedence than addition.

Memory Tip: [Comparison] Parentheses are "priority override buttons".

Q5. Type Matching Core

Pick the corrected statement for text "A":

A) string s = 'A'; B) char c = 'A'; C) char c = "A";

Answer: B is correct.

Why: Single quote is char; double quote is string literal.

Memory Tip: [Rule] Single quote = one char, double quote = text string.

Q6. Function Return Type Core

Why is this wrong: void f() { return 5; }?

Answer: void means no return value.

Fix: Use int f() or remove returned value.

Memory Tip: [Rule] void = "no package sent back".

Q7. cin/cout Direction Basic

Choose correct input statement:

A) cin << age; B) cin >> age;

Answer: B

Why: >> extracts data from input stream into variable.

Memory Tip: [Analogy] Arrow points toward variable for input.

Q8. Comprehensive Mini Debug Challenge

Find two bugs:

double average(int a, int b) {
    return (a + b) / 2;
}
int main() {
    int x;
    cin << x;
    cout << average(x, 5) << endl;
}

Answer: (1) cin >> x; should use extraction operator. (2) better use / 2.0 for decimal average.

double average(int a, int b) {
    return (a + b) / 2.0;
}
int main() {
    int x = 0;
    cin >> x;
    cout << average(x, 5) << endl;
    return 0;
}
Memory Tip: [Checklist] operator direction, initialization, numeric type.
