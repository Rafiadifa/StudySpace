Part 1: if Statements & Common Mistakes

Exercise 2.1 Basic

= vs == in conditions

What is wrong with this code? What will actually happen when it runs?

int x = 5; if (x = 10) cout << "x is 10" << endl;
Hint: In C++, = is assignment; == is comparison. The result of an assignment is the value assigned.
Solution

C is correct. Use if (x == 10) to test equality. With if (x = 10), x becomes 10 and the expression evaluates to 10 (non-zero = true), so the body always runs.

Exercise 2.2 Basic

Missing braces

If x is 0, which lines run?

if (x > 0) cout << "A" << endl; cout << "B" << endl;
Hint: Without braces, only the immediately following statement is part of the if. Indentation does not change behavior.
Solution

C. Only the first cout is inside the if. The second cout always runs. Use { } to group both statements.

Exercise 2.3 Basic

Semicolon after if

What does this program print when x is 5?

if (x > 0); { cout << "Positive" << endl; }
Hint: The semicolon after if (x > 0) ends the if statement. The block { ... } is then a normal block, not part of the if.
Solution

It always prints Positive. The ; makes the if body empty, so the block below runs unconditionally. Remove the semicolon to fix.

Exercise 2.4 Basic

Comparison operators

Which expression is true when we want "score is in the range 60 to 100 (inclusive)"?

Hint: "Between 60 and 100 inclusive" means score ≥ 60 and score ≤ 100. C++ does not support chained comparisons like 60 <= score <= 100 the way math does.
Solution

A. Use && to require both conditions. B is wrong (chained comparison evaluates differently); C would be true for any score; D excludes 60 and 100.

Part 2: if-else and if-else-if

Exercise 2.5 Basic

if-else: exactly one path

With score = 45, what is printed?

if (score >= 60) cout << "Pass" << endl; else cout << "Fail" << endl;
Solution

B) Fail. 45 < 60, so the else branch runs. In if-else, exactly one of the two blocks runs.

Exercise 2.6 Intermediate

if-else-if order (common bug)

What grade is printed when score = 95 with this buggy code?

if (score >= 60) cout << "D" << endl; else if (score >= 80) cout << "B" << endl; else if (score >= 90) cout << "A" << endl; else cout << "F" << endl;
Hint: Conditions are checked top to bottom. The first true condition wins. 95 >= 60 is true, so we never reach the 80 or 90 checks.
Solution

C) D. Order matters! Check from highest first: score >= 90, then 80, then 60, then else for F.

Exercise 2.7 Basic

Even or odd

What is the correct condition to check if n is even?

Solution

A. n % 2 is the remainder when n is divided by 2. Even numbers have remainder 0. Use == for comparison.

Part 3: switch Statement

Exercise 2.8 Intermediate

Missing break (fall-through)

What is the output when choice = 1?

switch (choice) { case 1: cout << "One" << endl; case 2: cout << "Two" << endl; case 3: cout << "Three" << endl; }
Hint: Without break, execution falls through to the next case. So after "One", it continues into case 2 and case 3.
Solution

C. Without break, control falls through: One, Two, Three. Add break; after each case to exit the switch.

Exercise 2.9 Basic

switch expression type

Which type can you use in switch (expression)?

Hint: switch works with integral types (int, char) and enums. It does not work with floating-point or string.
Solution

A. switch requires an integer-like expression (int, char, enum). Not double, float, or string. For strings use if-else-if.

Exercise 2.10 Basic

default case

When does the default case in a switch run?

Solution

B. default runs when the expression does not match any case value. Use it for invalid or unexpected values.

Part 4: Loops and Arrays

Exercise 2.11 Basic

for loop: how many iterations?

How many times does the loop body run?

for (int i = 0; i < 5; i++) cout << i;
Hint: i = 0,1,2,3,4 satisfy i < 5. When i becomes 5, the condition fails. So 5 iterations.
Solution

B) 5. i takes values 0,1,2,3,4. Use i < 5 for exactly 5 iterations (not i <= 5, which would be 6).

Exercise 2.12 Intermediate

Off-by-one error

We have int arr[5] = {10, 20, 30, 40, 50};. What does arr[5] refer to?

Hint: Valid indices for an array of size 5 are 0, 1, 2, 3, 4. The last element is arr[4].
Solution

C. Indices are 0..4. arr[5] is out of bounds (off-by-one). Last element is arr[4]. Accessing arr[5] is undefined behavior.

Exercise 2.13 Basic

break vs continue

What does this loop output?

for (int i = 0; i < 6; i++) { if (i == 3) break; cout << i << " "; }
Hint: break exits the loop entirely. So we print for i=0,1,2 and then break at i=3.
Solution

Output: 0 1 2 . When i is 3, break runs and the loop ends. So 3, 4, 5 are never printed.

Exercise 2.14 Basic

continue: skip iteration

What does this loop output?

for (int i = 0; i < 5; i++) { if (i % 2 == 0) continue; cout << i << " "; }
Hint: continue skips the rest of the current iteration and goes to the next. So we only print when i is odd (1, 3).
Solution

Output: 1 3 . When i is even (0,2,4), continue skips the cout. So only 1 and 3 are printed.

Exercise 2.15 Basic

while vs for

When is a while loop typically preferred over for?

Solution

B. Use while when you don't know how many times to repeat (e.g. "read until valid input"). Use for when you have a clear count or range.

Part 5: Logical Operators & Output Prediction

Exercise 2.16 Basic

Logical AND and OR

What is the value of (true && false) || true?

Hint: && is true only when both sides are true. || is true when at least one side is true. (true && false) is false; false || true is true.
Solution

B) true. (true && false) = false. false || true = true.

Exercise 2.17 Intermediate

Reverse number output

After this loop, what is the value of reversed?

int number = 123; int reversed = 0; while (number > 0) { int digit = number % 10; reversed = reversed * 10 + digit; number /= 10; }
Hint: Each iteration: digit = number % 10 (last digit), reversed = reversed*10 + digit, number = number/10. Trace: 3, then 32, then 321.
Solution

321. The loop strips digits from number (123 → 12 → 1 → 0) and builds reversed (0 → 3 → 32 → 321).

Exercise 2.18 Intermediate

Find maximum in array

What is the output? int arr[] = {3, 7, 2, 9, 1}; int size = 5;

int maxVal = arr[0]; for (int i = 1; i < size; i++) { if (arr[i] > maxVal) maxVal = arr[i]; } cout << maxVal;
Solution

9. The loop keeps the largest value seen so far in maxVal. The maximum in {3,7,2,9,1} is 9.

Exercise 2.19 Advanced

Loop output prediction

What does this program print?

int sum = 0; for (int i = 0; i <= 4; i++) sum += i; cout << sum;
Hint: i goes 0, 1, 2, 3, 4. sum = 0+1+2+3+4 = 10.
Solution

10. sum = 0+1+2+3+4 = 10. Note: i <= 4 gives five iterations (0..4).

Exercise 2.20 Intermediate

enum and switch

What is the first value in an enum if not specified?

enum Color { RED, GREEN, BLUE };
Solution

A) 0. RED=0, GREEN=1, BLUE=2 by default. Each subsequent value is one more unless you assign explicitly.
