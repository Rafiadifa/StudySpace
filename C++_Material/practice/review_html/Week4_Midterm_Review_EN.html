Chapter 3: Control Statements & Operators

Midterm Review (English) - Focus on execution flow and common traps

A. Exam Focus in 60 Seconds

  • Integer division truncates: 7/2 = 3, not 3.5! Use (double)a/b for decimals.
  • Prefix vs Postfix: ++x (increment first), x++ (use first). Know the difference!
  • Short-circuit: && stops if first is false, || stops if first is true.
  • switch fall-through: Missing break executes the next case!
  • Dangling else: else pairs with the nearest unmatched if - always use braces!
  • while vs do-while: while checks first (may skip), do-while checks last (runs at least once).
  • break vs continue: break exits the loop, continue skips to next iteration.

B. Operator Precedence & Traps

Top 5 Deadly Mistakes:
  1. if (x = 5) → Assignment, always true! Use == for comparison.
  2. 7 / 2 expecting 3.5 → Integer division truncates to 3.
  3. Missing i++ in loop → Infinite loop!
  4. Semicolon after if (...); → Empty statement, braces always execute!
  5. arr[5] for size-5 array → Off-by-one error, valid indices are 0-4!

C. Control Structures Quick Reference

if (condition) { // executes if true } else if (other) { // else-if ladder } else { // default }
Dangling Else Rule: else pairs with the nearest unmatched if. Always use braces to avoid ambiguity!
switch (expression) { case 1: // code break; // DON'T forget! case 2: // falls through if no break default: // catch-all }
Fall-through: Without break, execution continues to the next case! Useful for grouping (e.g., cases A, B, C all do the same thing), but dangerous if accidental.
// while: check first, may never execute while (condition) { // loop body // MUST modify condition variable! } // do-while: execute first, check last do { // runs at least once } while (condition);
Use while when you might need zero iterations. Use do-while when you need at least one execution (e.g., menu systems).
for (init; condition; increment) { // loop body } // Example: print 0-9 for (int i = 0; i < 10; i++) { cout << i << " "; } // Multiple variables with comma operator for (int i = 0, j = 9; i < j; i++, j--) { // process from both ends }
Best for: Known iteration count. Can use comma operator for multiple initializations/increments.

D. Execution Tracing Drills

Drill 1: Pre/Post Increment core

int x = 5, y = 5; int a = x++; // a = ?, x = ? int b = ++y; // b = ?, y = ? int c = a + b; // c = ?
a = 5 (use first), x = 6
b = 6 (increment first), y = 6
c = 11

Drill 2: Short-Circuit core

int a = 3, b = 5, c = 7; bool r1 = (a > b) && (++c > b); // c = ? bool r2 = (a < b) || (++c > b); // c = ?
After r1: c = 7 (&& short-circuits, ++c never executes)
After r2: c = 7 (|| short-circuits, ++c never executes)
Key: && stops if first is false, || stops if first is true.

Drill 3: Loop Output core

for (int i = 1; i <= 5; i++) { if (i == 2) continue; if (i == 4) break; cout << i << " "; }
Output: 1 3
i=1: printed
i=2: continue (skipped)
i=3: printed
i=4: break (loop ends)

Drill 4: switch Fall-Through basic

char grade = 'B'; switch (grade) { case 'A': cout << "Excellent "; case 'B': cout << "Good "; case 'C': cout << "Average "; break; default: cout << "Poor"; }
Output: Good Average
'B' matches, prints "Good ", falls through to 'C', prints "Average ", hits break.
