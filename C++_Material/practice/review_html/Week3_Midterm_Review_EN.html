Control Flow Midterm Review

Fast, beginner-friendly English review for students with about 2 months of C++ study.

A. Exam Focus in 60 Seconds

  • Master if / if-else / if-else-if ladder structures and condition ordering.
  • Know when to use switch (integer/char values) vs if-else-if (ranges).
  • Remember loop syntax: for (init; condition; increment) and while (condition).
  • Don't forget break and continue control statements.
  • Watch out for off-by-one errors and infinite loops.
  • Understand short-circuit evaluation for logical operators (&&, ||, !).

B. Concept Cheat Sheets

Structure When to Use Key Syntax
if Single condition if (condition) { ... }
if-else Two alternatives if (c) { } else { }
if-else-if Multiple ranges Check highest first!
switch Discrete values (int/char) case x: ... break;
Common trap: Order matters in if-else-if! Check highest values first.
Loop Best For Syntax Pattern
for Known iteration count for (int i=0; i<n; i++)
while Unknown count, condition-based while (condition)
do-while At least one execution do { } while (c);
Memory tip: for = counting; while = waiting; do-while = guaranteed run.
Operator Meaning Example
== Equal to (comparison) 5 == 5 // true
!= Not equal to 5 != 3 // true
&& Logical AND A && B (both must be true)
|| Logical OR A || B (either can be true)
! Logical NOT !true // false
% Modulo (remainder) 10 % 3 // 1
Common trap: = is assignment, == is comparison!
Statement What it Does Use Case
break Exit loop/switch immediately Found what we're looking for
continue Skip to next iteration Skip certain values
return Exit function Return result to caller
Analogy: break = emergency exit; continue = skip this one, next please.

C. Must-Know Code Templates

1. Grading with if-else-if (order matters!)

int score = 85;

if (score >= 90) {
    cout << "Grade A" << endl;
} else if (score >= 80) {    // 85 >= 80 is true!
    cout << "Grade B" << endl;  // This prints!
} else if (score >= 70) {
    cout << "Grade C" << endl;  // Skipped!
} else {
    cout << "Grade F" << endl;
}

2. Calculator Menu with switch

int choice = 2;
switch (choice) {
    case 1: cout << "Add"; break;
    case 2: cout << "Subtract"; break;  // Matches!
    case 3: cout << "Multiply"; break;
    default: cout << "Invalid"; break;
}

3. Array + for loop pattern

int scores[5] = {85, 92, 78, 88, 95};
int sum = 0;

// Standard array loop pattern
for (int i = 0; i < 5; i++) {
    sum += scores[i];
}
double avg = sum / 5.0;

4. Input validation with while

int number;
cout << "Enter positive number: ";
cin >> number;

while (number <= 0) {  // Repeat until valid
    cout << "Invalid! Try again: ";
    cin >> number;
}

C2. Comparison: == vs = (Common Trap)

Operator Purpose Example Result
= Assignment x = 5 Puts 5 in x
== Comparison x == 5 Returns true/false
Critical mistake: if (x = 5) assigns 5 to x and returns 5 (always true)!
Correct: if (x == 5) compares x with 5.

Another Common Trap: Semicolon after if

if (x > 0); {        // WRONG! Semicolon ends if
    cout << "x is positive";
}
The semicolon creates an empty if body. The braces create a separate block that always runs!

D. Hard Parts & Common Mistakes

Hard Point 1: Off-by-one errors in loops.
Real rule: Arrays are 0-indexed. Loop should be i < n not i <= n.
Hard Point 2: Wrong order in if-else-if ladder.
Real rule: Check most restrictive/highest values first. score >= 90 before score >= 60.
Hard Point 3: Forgetting break in switch.
Real rule: Without break, execution "falls through" to next case!
Hard Point 4: Infinite loops from missing increment.
Real rule: Always ensure loop variable moves toward termination condition.
Quick anti-mistake checklist: Check loop bounds, verify condition order, confirm break statements, trace variable updates.

E. Practice Zone

Q1. Condition Operator Basic

What's the difference between x = 5 and x == 5 in an if statement?

Answer: x = 5 assigns 5 to x (always true). x == 5 checks if x equals 5.

Why it matters: Using = in if condition always executes the body and changes your variable!

Memory Tip: = puts value in box, == checks if box has value.

Q2. if-else-if Order Core

What's wrong with this grading code?

if (score >= 60) cout << "D";
else if (score >= 70) cout << "C";
else if (score >= 80) cout << "B";
else if (score >= 90) cout << "A";

Answer: Wrong order! A score of 95 triggers the first condition (>= 60) and prints D.

Fix: Check from highest to lowest: 90, 80, 70, 60.

Memory Tip: Check from VIP to general - highest first!

Q3. Switch Fall-through Core

What does this code output?

int day = 1;
switch (day) {
    case 1: cout << "Monday ";
    case 2: cout << "Tuesday ";
    case 3: cout << "Wednesday ";
    default: cout << "Other";
}

Answer: "Monday Tuesday Wednesday Other"

Why: Without break, execution falls through all subsequent cases.

Memory Tip: switch is like a water slide - once you're in, you keep sliding until a break stops you.

Q4. Loop Off-by-One Core

For an array of 5 elements (indices 0-4), which loop is correct?

A) for (int i = 0; i <= 5; i++)
B) for (int i = 0; i < 5; i++)
C) for (int i = 1; i <= 5; i++)

Answer: B is correct.

Why: Arrays are 0-indexed. Valid indices: 0, 1, 2, 3, 4. Loop A goes to index 5 (out of bounds). Loop C misses index 0.

Memory Tip: 0-based: start at 0, use < (not <=), end at n-1.

Q5. Short-Circuit Evaluation Core

Why is this code safe from division by zero?

int n = 0;
if (n != 0 && 100 / n > 5) {
    cout << "Result is large";
}

Answer: Short-circuit evaluation. Since n != 0 is false, the second part 100 / n is never evaluated.

Key concept: && stops if left side is false. || stops if left side is true.

Memory Tip: && is like "stop light" - first false stops everything. Put safety checks first!

Q6. while vs do-while Basic

When would you use do-while instead of while?

Answer: Use do-while when the loop body must execute at least once.

Example: Menu display - show menu, get choice, repeat if invalid.

int choice;
do {
    cout << "1.Play 2.Exit\n";
    cin >> choice;
} while (choice != 2);
Memory Tip: while = "maybe zero times"; do-while = "at least once".

Q7. Find the Bug (Infinite Loop) Core

int x = 0;
while (x < 10) {
    cout << x << endl;
    // forgot x++ here!
}

What's wrong and how to fix it?

Answer: Infinite loop! x never changes, so x < 10 is always true.

Fix: Add x++; inside the loop body.

Memory Tip: Every loop needs: (1) condition, (2) body, (3) progress toward ending.

Q8. Number Reversal Algorithm Challenge

Trace this code with number = 1234:

int number = 1234;
int reversed = 0;
while (number > 0) {
    int digit = number % 10;
    reversed = reversed * 10 + digit;
    number /= 10;
}

What is the final value of reversed?

Answer: 4321

Trace:

  • Iter 1: digit=4, reversed=4, number=123
  • Iter 2: digit=3, reversed=43, number=12
  • Iter 3: digit=2, reversed=432, number=1
  • Iter 4: digit=1, reversed=4321, number=0
Memory Tip: % 10 gets last digit, / 10 removes last digit, * 10 shifts left to make room.
